Kommunikationsobjekt-Tabelle:

1.Byte:     Anzahl Objekte (a 3 Byte)
2.Byte:     Zeiger auf RAM-Flag Tabelle (braucht man  nicht wirklich)
ab 3. Byte folgen die Objektbeschreibungen, immer 3 Byte pro Objekt:
        1.Byte: Zeiger auf Wert
        2.Byte: Konfigurations-Byte:
            Bit7:   immer 1
            Bit6:   Transmission enable (0=no transmission)
            Bit5:   Value Memory type (0=RAM, 1=EEPROM)
            Bit4:   Write enable (1=enable)
            Bit3:   Read enable (1=enable)
            Bit2:   Communication enable (1=enable)
            Bit1 und 0: Transmission Priority
                11=low operational priority
                10=high operational priority
                01=alarm priority
                00=system priority
        3.Byte: Typ-Byte:
            0=1 Bit
            1=2 Bit
            2=3 Bit
            3=4 Bit
            4=5 Bit
            5=6 Bit
            6=7 Bit
            7=1 Byte
            8=2 Byte
            9=3 Byte
            10=4 Byte
            11=6 Byte
            12=8 Byte
            13=10 Byte
            14=14 Byte

Der "NodeParam" Bereich wurde für RF um 16 + 2 byte aufgebohrt.
(Folgende Adressen sind mit dem BASE_ADDRESS_OFFSET von 0x100 beaufschlagt,
damit man sie direkt mit mread/mwrite verwenden kann)
0x0100-0x01ff  : alter NodeParam Bereich

0x0200       :  RSSI (Received Signal Strength Indication)

0x0208       :  forward PA. Messages with physical dest address are forwarded if upper byte of
                dest address matches. Only the bits that are 1 in the MASK must match (see below).
0x0209       :  MASK for the forward PA.

0x020a-0x0120f   serial number / domain address (6 byte)

0x0210-0x0211 :  Abgleichwert für den Oszillator des RFM22.
